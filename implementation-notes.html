Hermes Desktop nick Launcher Implementation Notes

  • Created the local wrapper at ~/.local/bin/hermes-desktop-nick, the private local env at ~/.config/hermes-desktop-nick/env, and the Walker desktop entry at ~/.local/share/applications/hermes-agent-nick.desktop.
  • Created and enabled the remote user service hermes-desktop-remote.service on nick, bound to 0.0.0.0:9120 with --tui --insecure.
  • Fast-forwarded nick's Hermes checkout from 0.15.0 to 0.15.1 because the older remote code ignored HERMES_DASHBOARD_SESSION_TOKEN and generated an ephemeral session token.
  • Validation: protected /api/config returned HTTP 200 with the pinned token, Elephant returned hermes-agent-nick.desktop, and relaunching both profiles did not create extra Electron parent processes.
  • Desktop logs remain shared at ~/.hermes/logs/desktop.log because the alternate launcher intentionally sets HERMES_HOME=/home/shuv/.hermes to reuse the existing install.
